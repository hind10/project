Development

Environment

BusLine packages require a package manager to be installed. You may use npm to serve this purpose. A side note about node version: Please use LTS version(not Current) since some of the plugins does not work with it yet. NPM Installation: https://nodejs.org/en/download/

Creating a Next.js project

Create a new folder for your project and initialize it as an npm or yarn project with the following command. 

                            npm init -y  
    or
yarn init -y

Once this is done, you can install Next.js as well as dependencies, react and react-dom

                            
                                npm i --save react react-dom next
    or
yarn add --save react react-dom next

update your package.json to add the following start scripts: 

                            
                                
"scripts": {
    "dev": "next",
    "build": "next build",
    "start": "next start"
},

Installing Packages

Please navigate to project directory with command line and use one of the commands below. 

                        
                            npm i
  or
yarn

Starting Project

Runs the app in development mode. Open http://localhost:3000 to view it in the browser. The page will automatically reload if you make changes to the code. You will see the build errors and lint warnings in the console. 

                        
                            npm run dev
    or
yarn run dev

Creating a Production Build

When you’re ready to deploy to production, running npm run build will create an optimized build of your app in the build folder.

Your app is ready to be deployed.

                        
                            npm run build
    or
yarn run build

Deployment

npm run build or yarn run build creates a build directory with a production build of your app.

Set up your favorite HTTP server so that a visitor to your site is served index.html and requests to static paths like 

/static/js/main.><hash>.js
are served with the contents of the 

/static/js/main.<hash>.js
file.

For more information about different server configurations please see cra deployment guilde.

Libraries

Next

Next js is a JavaScript framework created by Zeit. It lets you build server-side rendering and static web applications using React next js.

react

A JavaScript library for building user interfaces: react.

react-feather

react-feather is a collection of simply beautiful open source icons for React.js. Each icon is designed on a 24x24 grid with an emphasis on simplicity, consistency and readability. react-feather.

reactstrap

Easy to use React Bootstrap 4 components reactstrap.

react-slick

React slick is a carousel component built with React. It is a react port of slick carousel react-slic.

react-anchor-link-smooth-scroll

scroll different sections in home page based react-anchor-link-smooth-scroll.

react-countup

Layout

/layout

There are one different main layout for src/pages/index.js and these are located at src/index.js file.

/containers

These folder contains smart components that may contain logic, view and other components. They are divided into logical sub folders providing a hint about their usage place. You may also create new ones or modify existing containers for your project needs.

/components

These folder contains dumb components which are mostly made to provide view. They receive their data from outside of themselves and used throughout the project and in containers. These are also divided into sub folders.

/data

In the data folder, you will find the specific data that are used on that specific section

/pages

Here are the pages (also known as views) of the application, each file will automatically match a route as described in Next.js documentation.

The _document.js is a special file which will only be used Server Side to render the basic HTML of the application.
